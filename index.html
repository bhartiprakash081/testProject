Learning to use git commands like:

  • Status :

    The status command in git is used to check the current status of the repo. It shows whether there are changes to be staged and/or comitted. This command is helpful as we use it to check the current status of our repo, before going on about anything.
    When combined with the diff tool, the status command becomes very helpful and crucial to development, as it lets us see and choose what changes to be comitted, and which changes shouldn't be committed.

  • Init :

    The git init command is the command that is used before anything else in a repo. The init command initializes the repository in the given folder. The init command tells git to do all the necessary requirements for git to track and manage the repo.
    This command is required only once in the lifetime of a repo.

  • Commit :

    The commit command is used to tell git to save the changes. It is kind of like telling git that okay, these are the changes that I am happy with, let's keep a reference to them.
    You generally would like to do a commit before leaving your repo, or changing branch to make sure that you don't loose any changes.
